ü™® The Stone: Data Foundations at Scale

You know variables, types, and basic data structures. But do you understand them at production scale? Let's level up.

Python - Production Patterns 
# Junior code: works but fragile
def get_user(user_id):
    user = db.query(f"SELECT * FROM users WHERE id = {user_id}")
    return user

# Senior code: secure, typed, documented
from typing import Optional
from dataclasses import dataclass

@dataclass
class User:
    """Represents a user in the system."""
    id: int
    name: str
    email: str
    is_active: bool = True

def get_user(user_id: int) -> Optional[User]:
    """
    Retrieve a user by ID.
    
    Args:
        user_id: The unique identifier of the user.
        
    Returns:
        User object if found, None otherwise.
        
    Raises:
        ValueError: If user_id is negative.
    """
    if user_id < 0:
        raise ValueError("user_id must be positive")
    
    # Parameterized query prevents SQL injection
    result = db.query("SELECT * FROM users WHERE id = %s", (user_id,))
    return User(**result) if result else None

üí° Key Differences: Type hints, dataclasses, docstrings, input validation, SQL injection prevention. This is what code review catches.

Pick a codebase you work on (or an open-source project). Analyze one module through the lens of a senior engineer. Document what you find and propose improvements.

üîç

Audit Checklist

  1. Map the system
    Create a component diagram of the module you're analyzing.
  2. Identify data flows
    What data goes in? What comes out? Create a sequence diagram.
  3. Spot code smells
    Ask Cursor: "Review this code for potential improvements"
  4. Propose one improvement
    Don't fix everything ‚Äî focus on the highest-impact change.
  5. Document your findings
    Write a brief technical memo as if presenting to your team.
