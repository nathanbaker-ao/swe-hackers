Architecture Decision Record (ADR) 
# ADR-0001: Use Event-Driven Architecture for Order Processing

## Status
Proposed

## Context
Our monolithic order processing system is becoming a bottleneck. 
Peak load during sales causes timeouts and failed orders.

## Decision
We will adopt an event-driven architecture using:
- Apache Kafka for event streaming
- Event sourcing for order state
- CQRS for read/write separation

## Consequences
Positive:
- Horizontal scalability
- Better failure isolation
- Audit trail via event log

Negative:
- Increased operational complexity
- Eventual consistency requires handling
- Team needs Kafka training
