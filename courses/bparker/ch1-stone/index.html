üß† Mental Models: Variables as Memory

You remember int x = 5; from C#. You remember let x = 5; from JavaScript. Now in Python, it's just x = 5. But beneath the syntax, the same thing is happening:

A variable is a named location in memory. Think of your brain: when someone says "your childhood home," you immediately recall an address, an image, feelings. The word triggers the memory. Variables are the same ‚Äî names that trigger memory retrieval.

üí° Key Insight

Python is dynamically typed like JavaScript, not statically typed like C#. You don't declare types ‚Äî Python figures them out at runtime. This feels liberating, but you trade compile-time error checking for runtime errors. Tools like Cursor's AI can help catch these issues.

Functions

Python 
# Function definition - def instead of function or void
def greet(name: str) -> str:
    """Return a greeting message."""
    return f"Hello, {name}!"

# Type hints are optional but helpful
def add(a, b):
    return a + b

# Default arguments
def power(base, exponent=2):
    return base ** exponent

# Using the functions
print(greet("Brittany"))  # Hello, Brittany!
print(power(3))           # 9 (3^2)
print(power(3, 3))        # 27 (3^3)

üí° Note: Python uses indentation instead of braces {}. This is the biggest adjustment from C#/JS. 4 spaces is standard.

In Python, everything is an object. When you write age = 30, you're not putting the number 30 into a variable. You're creating an integer object with value 30, and making age point to it. This is why Python is called a "reference-based" language.

Python 
# Immutable: Creates new object, doesn't modify
x = 5
y = x       # y points to same object as x
x = x + 1   # x now points to NEW object (6)
print(y)    # Still 5! y wasn't affected

# Mutable: Modifies the same object
a = [1, 2, 3]
b = a       # b points to SAME list as a
a.append(4) # Modifies the list in place
print(b)    # [1, 2, 3, 4] - b sees the change!

# To avoid this, make a copy
c = a.copy()  # or a[:] or list(a)

‚ö†Ô∏è Classic Bug: Accidentally modifying a shared mutable object. If you pass a list to a function and modify it, the caller sees those changes!
